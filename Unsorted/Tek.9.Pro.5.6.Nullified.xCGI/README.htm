Setup instructions for LINE9.com - Tek9 Pro v5.6

Step 1: Setup the Server

  1. If you are hosting your software with LINE9, then..
    1. Extract all files from the .zip onto your workstation, and open the \db\database.mdb file. Open the Table called TBLGLOBAL. Edit the field in this Table called HOMEPAGE_URL. The default value in this field is set to an IP address of 127.0.0.1, but you will have to edit this to the IP address or domain name for your site. example: The HOMEPAGE_URL field could contain: 127.0.0.1, or: myDomain.net, or: line9.com, or: machinename/cart, etc etc. Get the idea? If not, click here.
    2. Next, we have to edit the DSN in the \inx\database_connect.inc file. This is the one file which connects your entire application to the database itself. Our SysAdmin has created the DSN at the server for you, and set it to the name of your domain. Just change the mydsn name that's in that file now, to the name of your domain..
      example: If the name of your domain is: yourdomain.com, then the name of your DSN will be: yourdomain
      The code in database_connect.inc will look similar to this: DSNString = "DSN=yourdomain"
    3. Ok, now that's all set, upload all these files and folders (but NOT the \db\ folder!) into your webroot. This is the default directory that you get logged in to when you use your FTP Client (such as CuteFTP, or Ws_FTP) to login to our servers. Make sure you maintain the directory structure that the .zip file extracts it's files/folders to! example: \ftproot\, \img\, \inx\, \ssl\, etc. All of these sub-directories MUST reside under your webroot, just as the .zip file created them for you.
    4. NOW, upload your database.mdb file, and the merchant_conf file to your /dbases/yourdomain.com/ location. (our SysAdmin will give you complete instructions on how to use your FTP client to login and upload your files, and database).
    5. Now that you have uploaded your database to its secure place on the server, delete the empty \db\ folder off your workstation.

  2. If you are hosting your software yourself (on your own webserver) then..
    1. At your workstation; Extract all files from the .zip onto your workstation, and open the \db\database.mdb file. Open the Table called TBLGLOBAL. Edit the field in this Table called HOMEPAGE_URL. The default value in this field is set to an IP address of 127.0.0.1, but you will have to edit this to the IP address or domain name for your site. example: The HOMEPAGE_URL field could contain: 127.0.0.1, or: myDomain.net, or: line9.com, or: machinename/cart, etc etc. Get the idea? If not, click here.
    2. At your workstation; Next, we have to edit the DSN in the \inx\database_connect.inc file. This is the one file which connects your entire application to the database itself. We will create the actual DSN at the server in a moment. Just change the mydsn name that's in that file now, to the name of your domain..
      example: If the name of your domain is: yourdomain.com, then the name of your DSN will be: yourdomain
      The code in database_connect.inc will look similar to this: DSNString = "DSN=yourdomain"
    3. At the server; Share the wwwroot folder (or whatever folder name you gave to your webroot when you installed Internet Information Server (IIS)) so that you can upload files to it from your workstation.
    4. At the server; Next, Set the Default Document(s);
      • If your webserver Operating System is..
      • Windows 2000 Server, open your Microsoft Management Console (MMC) and go to the Advanced properties for your Default Website.
      • Windows NT Server (with SP6a), then open the Internet Services Manager, and go to the Properties for your Default Website. The Advanced tab will allow you to enter your Default Documents.
      • NOW that you have found the Default Document(s) field, just enter the following string:
        tek9.asp, default.asp, default.htm (these are the default files which your server will look for first, if you just enter a blank website address such as: www.line9.com/ into your web browser. (the server has to know what file your browser is asking for!))

    5. At the server; Next, we need to create a secure location on the webserver for your database file (other than the webroot). example:
      C:\InetSecure\databases\yourdomain.com\ (Why do this? So that visitors to your website cannot download your database file right over the web!
    6. At the server; Now we have to set the security permissions on the new folder you just created: Open Windows Explorer, right-click on the folder where your database will be located, click "Properties". Click security, and remove the "Everyone" group from the current permissions. Click "Add" and select the IUSR_machinename account and give that user Modify/Change permissions. (this will allow users to add items to their cart), and do the same thing for the IWAM_machinename account. Allow these new permissions to propogate to all files and sub-folders. Click OK.
    7. At the server; Share this new directory over the network too, so that you can upload your database to it from your workstation.
    8. At your workstation; Ok, now that's all set, upload all these files and folders (but NOT the \db\ folder!) into your webroot at the server. Make sure you maintain the directory structure that the .zip file extracted its files/folders to! example: \ftproot\, \img\, \inx\, \ssl\, etc. All of these sub-directories MUST exist, and reside under your webroot, just as the .zip file created them for you.
    9. At your workstation; NOW, upload your database.mdb file, and the merchant_conf file to your new, secured database directory.
    10. At your workstation; Now that you have uploaded your database to its secure place on the server, delete the empty \db\ folder off your workstation.
    11. At the server; Install the object(s) that you will want to use to enhance this software. None of these are required, as your software will operate just fine without them, but real-time services, gateways, and subscription based vendors require servers to use their objects. These objects are what enable your webserver to communicate with theirs.
      • \_objects\Cybercash\ - Only needed if you use real-time Cybercash gateway to process your credit cards.
      • \_objects\FedExapi\ - Only needed if you want real-time FedEx integration on your Checkout page.
      • \_objects\MDAC\ - Only needed if you experience database or ODBC problems on your server.
      • \_objects\MightyUp\ - Only needed if you want to utilize web-based file upload.. No more FTP!
      • If you can't use MightyUp, then the webserver must have Microsoft Internet Explorer (IE) v5.0 or later.
      • \_objects\MightyZip\ - Requires subscription with USPS zip lookup service. Auto-fills (City/State) on your Checkout page.
      • \_objects\PayflowPro\ - Only needed if you use real-time Verisign Payflow Pro gateway to process your credit cards.
      • \_objects\UPSapi\ - Only needed if you want real-time UPS integration on your Checkout page.
      • \_objects\USPSapi\ - Only needed if you want real-time USPS(Postal Service) integration on your Checkout page.
    12. At the server; Next, we need to connect your database.mdb file to your application; The application calls your database by name, so to do this we:
      Create a Microsoft Access System DSN, and point it at your secured database.mdb file on the webserver.. (Start, Settings, Control Panel, ODBC Data Sources), and name the new DSN: yourdomain (or whatever name you gave to it in your database_connect.inc file). By the way, it's Case Sensitive, so be careful!) After you have typed your Data Source Name, browse to, and select the database.mdb file on the server. All you have to do is double-click on the the .mdb filename, and click OK. Leave everything else set to their defaults. This makes it so that your application software can refer to your database by name. In this case, the 'name' of your database is now: yourdomain
    13. Now, get back to your workstation, it's time to start on your site! :)

Step 2: Edit your template and website style file

  1. The file you will want to edit to make this software conform with your existing website resides at \inx\template.inc. This file acts as the template, or framework for the pages in your site. There are instructions within that file too, which show you how to integrate the Template with your site.
  2. Lastly, edit your website style file: \inx\style.inc. We have included some example JavaScripts and StyleSheet code if you wish to use it, otherwise just delete what's there and add your own.

Step 3: Login to your Intranet, and start adding pages and inventory

  1. Use a fairly new web browser such as Internet Explorer v5.0 or later, free from Microsoft.
  2. We have included a web-based Intranet for you to be able to login and make changes;
    (much easier than trying to manipulate individual files).
    Login to: http://machinename/ssl/intranet/ (this loads your Default Document, whether running this software locally, or remotely)
    username: test
    password: test
  3. (The above login username and password can be edited thru your Intranet also (in the Contacts Editor), OR, directly in your database thru tblContacts).
