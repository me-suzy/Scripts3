Aide du WebFTP

Boîte de connexion

Pour utiliser le WebFTP, vous devez d'abord vous identifier.
Pour cela, vous devez entrer vos informations de connexion dans les champs appropriés :

Nom d'utilisateur :

C'est le nom de votre compte FTP fourni par votre administrateur ou votre hébergeur.
Attention, souvenez vous que certains serveurs FTP sont sensibles à la casse, autrement dit
les noms « utilisateur » et « UtIlIsaTeUR » sont différents.

Mot de passe :

C'est le mot de passe associé à votre nom de compte que vous avez choisi ou qui vous.
a été transmis lors de votre inscription. Souvenez vous aussi de la casse des lettres,
les mots de passe « passe » et « PaSsE » sont différents.

Serveur et port :

Cette option n'est pas toujours disponible mais vous permet de choisir à quel serveur
FTP vous souhaitez vous connecter. Ce peut être soit un nom de type « ftp.serveur.com »
ou une adresse IP de type 000.000.000.000

Dossier :

Si cette option est définie, le script entrera dans le dossier fourni lors de la connexion.
Si vous ne savez pas quoi mettre, laissez la valeur par défaut.

Temps maximum :

Si ce champ existe, il vous permet de déterminer le temps maximum d'attente
de réponse du serveur, en secondes. Après ce délai, la connexion FTP échoue.

Mode de transfert :

Si ce champ existe, il vous permet de définir le mode de transfert à utiliser pour
les téléchargements. Le mode ASCII ne s'applique que pour les fichiers de type texte
(html, php, javascript), sinon il faut utiliser le mode binaire par défaut.

Mode FTP passif :

Si ce champ existe, il vous permet d'activer le FTP passif, qui peut être nécéssaire
avec certains serveurs. Si par exemple la connexion échoue sur un serveur existant, essayez
d'activer cette option.

Connexion avec SSL :

Si ce champ existe, il vous permet de vous connecter à des serveurs FTP sécurisés.
Notez que dans ce cas, le port de connexion pourrait être différent.

Boutons de l'interface

La gestion des fichiers se fait grâce aux icônes dans la barre d'action,
située sous la liste des fichiers. Voici la description de ces bouton :

Suppression de fichiers :

Une fois que vous avez séléctionné les fichiers avec les cases à cocher,
cliquez sur ce bouton pour les supprimer. Le script vous demande une confirmation
car tous les sous dossiers seront aussi supprimés.

Renommer les fichiers :

Une fois que vous avez séléctionné les fichiers avec les cases à cocher,
cliquez sur ce bouton pour les renommer. Pour chaque fichier ou dossier,
le script vous demande le nouveau nom.

Déplacer les fichiers :

Une fois que vous avez séléctionné les fichiers avec les cases à cocher,
cliquez sur ce bouton pour les déplacer vers un autre endroit. Le script vous
demande le dossier de destination, il peut être absolu (/home/user/f) ou
relatif (sousdossier/). Si ce dossier n'existe pas, il sera créé.

Nouveau dossier :

Le script vous demande le nom du nouveau dossier; il peut être absolu
ou relatif. Afficher ensuite le résultat de la création.

Affichage et edition de fichier :

Ce bouton a un comportement particulier. Une fois que vous aurez cliqué dessus,
les cases à cocher pour la séléction se transforment en cases uniques (radio).
Choisissez ensuite le fichier à voir ou éditer, et cliquez encore sur ce bouton.
Une fenêtre s'ouvre et vous demande comment ouvrir le fichier.

Permissions :

Une fois que vous avez séléctionné les fichiers avec les cases à cocher,
cliquez sur ce bouton pour changer les permissions (CHMOD). Le script vous
demande alors la valeur des perissions, en octal. Consultez la page du
manuel PHP sur les permissions.

Création d'archive :

Ce bouton n'est pas toujours disponible selon la configuration.
Une fois que vous avez séléctionné les fichiers avec les cases à cocher,
cliquez sur ce bouton pour en faire un archive TAR qui est ensuite envoyée.
Les archives obtenues peuvent être compréssées avec GZIP ou BZIP2 si les
librairies sont présentes sur le serveur.

Envoi de commande SITE :

Ce bouton ne dépend pas des fichiers séléctionnés. Le script vous
demande la commande SITE à envoyer, et elle est transmise au serveur
Le script affiche si la commande a réussie ou échouée, mais on ne peut
pas récupérer la réponse du serveur.
