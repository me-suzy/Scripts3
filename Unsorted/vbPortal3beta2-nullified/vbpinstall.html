Base Requirements

To install vbPortal version 3.0 the following pre-requisites are necessary.

  • vBulletin version 2.2.4

If you don't own vBulletin, you may purchase it through this link Purchase vBulletin
or you can go to www.phpportals.com and click on the vBulletin banner to purchase.

Installing vbPortal (Standard Installation)

Current Version is vbPortal30pr8_1.zip

These instructions will cover the standard installation of vbPortal and the directory placement of vBulletin. The instructions and examples used in this document are based on default web server settings for Windows (NT, 2000) and Linux (RedHat 7.0)

Terms:
public_html: public_html is used to describe the path to your web site directory. This is the file path and not the URL.

Example (Windows): "C:\inetpub\wwwroot\"
Example (Linux): "/var/www/html/"

Note: If you have already installed vBulletin please make a backup of your database before continuing the installation. If vBulletin is not in a sub-directory it is recommended that you move it into a sub-directory as listed below.

If you are NOT installing vbPortal into a sub-directory please skip to the
"Install vBulletin (vBulletin directory placement)" section below

Installing vbPortal (in a sub-directory)
It is possible to install vbPortal into a sub-directory under your web server. You can use any sub-directory name that you would like. In this document vbportal will be used as the default sub-directory.

Example (Windows): "C:\inetpub\wwwroot\vbportal\"
Example (Linux): "/var/www/html/vbportal/"
URL = http://www.yoursite.com/vbportal/index.php (start page for vbPortal)

Note: Keep in mind, in the following examples you will need to add your sub-directory name into the appropriate place.

Forum sub-directory:
Example (Windows): "C:\inetpub\wwwroot\vbportal\forums\"
Example (Linux): "/var/www/html/vbportal/forums/"

config.php file:
Example (Windows):
$nukeurl = "http://www.yoursite.com";
$bburl = "http://www.yoursite.com/vbportal/forums";
$nukepath = "C:/inetpub/wwwroot/vbportal";
$vbpath = "C:/inetpub/wwwroot/vbportal/forums";

Example (Linux):
$nukeurl = "http://www.yoursite.com";
$bburl = "http://www.yoursite.com/vbportal/forums";
$nukepath = "/var/www/html/vbportal";
$vbpath = "/var/www/html/vbportal/forums";

Additional Note: Additional modifications will be necessary if you are installing into a sub-directory. These modifications will be covered at the end of this file, under Installing vbPortal, sub-directory, modifications

Install vBulletin (vBulletin directory placement)

  1. vBulletin should be installed under a sub-directory of your web site. The default sub-directory name is forums. You can create any sub-directory name that you like, just make sure you replace your sub-directory name with the default name "forums".

    Example (Windows): "C:\inetpub\wwwroot\forums\"
    Example (Linux): "/var/www/html/forums/"

  2. Complete the installation of vBulletin. The installation of vBulletin is beyond the scope of this document.
  3. Make sure that vBulletin has been installed and configured correctly before continuing.

Create New Forums

  1. Create two new forums.
    Use the vBulletin admin control panel to create these new forums. One of these forums will be used for vbPortal's front page news section and the second forum will be for the vbPortal's front page polls section. You can name these forums whatever you like.

  2. After creating these two new forums hold your mouse pointer over the link of each forum and write down the forumid number for each. The forumid numbers will be used in the vbPortal's config.php file to configure the front page news and polls sections.

Upload and Install vbPortal files

  1. Unzip the vbPortal zip file to a temporary directory on your computer.

    Example (Windows): C:\temp\vbPortal\
    Example (Linux): /tmp/vbPortal/

  2. Go into the temporary vbPortal "public_html" directory (cd C:\temp\vbPortal\public_html\). Copy all the files including the directories (DO NOT CHANGE THE DIRECORIES LAYOUT) to your web servers public_html directory (see public_html term above).

    Example (Windows): copy files and directories from "C:\temp\vbPortal\public_html\" to "C:\inetpub\wwwroot\"
    Example (Linux): copy files and directories from "/tmp/vbPortal/public_html/" to "/var/www/html/"

    Note: During the copying process you may be prompted to overwrite some files. These are gif files that are used for the main image and buttons. By default it's best just to overwrite these gif files.

Access vbPortal install file

  1. Through a web browser access vbPortal's install file which is located under forums/admin/vbpinstall.php.

    Example: http://www.yoursite.com/forums/admin/vbpinstall.php

  2. Follow through the vbPortal install script and select which option you would like to install.

  3. At the "Click here for instructions on the vB script mods needed." Click on the link and follow the instructions. Some of these modifications are required and some are optional. Please read through this section carefully.

  4. There is an additional section that covers TOPICS. If you selected the Full Install or Installed TOPICS you will need to make modifications to several of the vBulletin php files. Make sure that the modifications are done correctly. To access this file again it is located under forums/admin/vbpscripts.html.

Upload/Import and Configure vbPortal Default Style

  1. Make a back of your existing default vBulletin style set by using the vBulletin admin control panel, Styles, Download/Upload option.

  2. Depending on your version of vBulletin, (2.2.4, 2.2.5 or 2.2.6) open the Default22x.style.style file in a text editor. The file is located in the temporary directory under the style directory.

    Example (Windows): C:\temp\vbPortal\style\Default226.style
    Example (Linux): /tmp/vbPortal/style/Default226.style

    Look at the very first line.
    If you are using vBulletin version 2.2.1 it should look like this 2.2.1|||
    If you are using vBulletin version 2.2.0 it should look like this 2.2.0|||

    Make the change to fit your version of vBulletin.(DO NOT CHANGE ANYTHING ELSE, unless you know what you are doing.)

  3. Upload the Default22x.style set file by using the same Download/Upload option. When uploading the Default22x.style set choose to overwrite the "Default" style.

    Note: If you are having trouble uploading the Default22x.style copy the Default22x.style file to the forums/admin/ directory. Use the vBulletin admin control panel, Styles, Download/Upload, Import existing style set (from local computer) option. Change the Style file: (local) name to "Default22x.style". If you are still having trouble, close all browser windows and try this process again. This mainly applies to Windows servers.

  4. After uploading the Default22x.style correctly you will need to modify a setting under Styles, Modify, Default, [font/colors/etc], Image paths. Look for "http://www.yoursite.com/forums/images" and change it to match your web site. Save your changes.

  5. Now click on Replacements, Modify, Default, expand list. Under Custom Replacements look for this
    "images/smilies is replaced by http://www.yoursite.com/forums/images/smilies".
    Click on edit and change the www.yoursite.com to match your web site. Save your changes.

Configure vbPortal's config.php file

  1. Change to the public_html (not the temporary directory) directory and rename or copy the config.php.txt file to config.php. Open the config.php file in a text editor. The most important section of the config.php file is the General Site Configuration section. The variables listed below must be configured correctly. If they are not correct then your site will not be displayed correctly.

    Default settings
    $nukeurl = "http://www.yoursite.com";// This is the URL to your vbportal web site
    $bburl = "http://www.yoursite.com/forums";// This is the URL to your forums web site
    $nukepath = "/home/public_html";// This is the file path to your web directory
    $vbpath = "/home/public_html/forums";// This is the file path to your forums directory

    Note: Windows users may need to use forward slashes instead of back slashes
    Example (Windows):
    $nukeurl = "http://www.yoursite.com";
    $bburl = "http://www.yoursite.com/forums";
    $nukepath = "C:/inetpub/wwwroot";
    $vbpath = "C:/inetpub/wwwroot/forums";

    Example (Linux):
    $nukeurl = "http://www.yoursite.com";
    $bburl = "http://www.yoursite.com/forums";
    $nukepath = "/var/www/html";
    $vbpath = "/var/www/html/forums";

    Make the necessary changes to match your web server.

  2. Under The News Config section change
    $newsforum=3;
    to match the forumid number of the news forum that you created earlier.
    Do the same for
    $pollsforums=4;

  3. If you did a Full Install or installed the TOPICS option find
    $hastopics=0; and change it to
    $hastopics=1;
    This will enable the Topics option.

  4. Make any additional changes to the config.php file to suit your needs.

  5. Save the changes made to the config.php file.
That's it.

Installing vbPortal, sub-directory, modifications
  1. Modify "Main Menu" links
    Access and login to the vbPortal Administration section. http://www.yoursite.com/vbportal/admin.php
    Click on Side Blocks, under Main Menu click on the edit block icon. (Looks like magnifying glass & paper)
    Basically, remove the "/" from the a href=" ..." so it looks like this
    a href="index.php">Home<....
    Do the same thing to the rest of the Main Menu Links
    You will also need to do this to the Administration Block also.

  2. Open the config.php file in a text editor. Look for the $foot1 variable. Find the
    img src=\"/images/vbp.gif\" and change it to
    img src=\"$nukeurl/images/vbp.gif\"

  3. Open includes/blocks/admin.php in a text editor. Look for this line
    global $bburl,$DB_site,$admin,$prefix,$newsforum,$block_sidetemplate;
    change it to
    global $bburl,$DB_site,$admin,$prefix,$newsforum,$block_sidetemplate,$nukeurl;

    Find this
    a href=\"admin.php?op=reviews\">"._WREVIEWS."
    change it to this a href=\"$nukeurl/admin.php?op=reviews\">"._WREVIEWS."

    Find this
    a href=\"admin.php?op=links\">"._WLINKS."
    change it to this a href=\"$nukeurl/admin.php?op=links\">"._WLINKS."

That's it for the Installing vbPortal, sub-directory, modifications section.
