Using MySQL

To run Wimpy from MySQL, you will first need to edit the MySQL configuration variables that are located in the main wimpy.php file in the "MySQL setup section" The following variables need to be set according to your MySQL server settings:

$host = "localhost"; (your host)
$db = "mp3s"; (the database you would like to use... you will have to set this up yourself)
$table = "wimpy"; (the name of the table you would like to use... wimpy will create a new table for you if you so desire)
$user = "username"; (your username for the database)
$pwd = "1234"; (your password for the database)

The MySQL connection files that are located in the "goodies" folder must be moved to your main wimpy folder and reside next to wimpy.php. This folder should also contain all of your mp3s. Wimpy will not read recursively into directories, so you will need to make sure that your mp3 files are all located in the same directory as wimpy.php

If you already have an mp3 database, you can configure Wimpy to run from it, however, there are a couple of required fields that need to be sent into wimpy (You should have a column in your database named "filename"). It is recommended That you first test out wimpy by letting wimpy create a new table on your database.

To have Wimpy set up a table on your database, access the mysql_update.php file. Wimpy will add all of the mp3 files that are located in your wimpy folder. This procedure is also a good way to make sure that your connection username, database and password are set up correctly.

Next, test wimpy by accessing the text.php?with a query string.
Example 1:
test.php?queryWhere=artist&queryValue=coldplay
The two variables that you need to pass in to test.php are queryWhere and queryValue.

queryWhere = the column you want to search your database on.

queryValue = the value that you are searching for. If you want to search for more that one value, then you would seperate|your requests|with|pipes. (It would probably be best to url encode%020your%020requests%020too (not the pipes, just the requests).
Example 2:
test.php?queryWhere=artist&queryValue=coldplay|the%020beatles
(there is a pipe "|" between coldplay | and | the beatles)
The example above would return al of the information from your wimpy table on your database where artist was equal to "coldplay" and "the beatles"

The code that is generated needs to go into a PHP page. An example "test.php" file is included in the MySQL Connection folder.

You can use the Wimpy customization tool (www.wimpyplayer.com/customize) to generate the code required to run wimpy. Obviously you will set the" Are you running Wimpy from a MySQL database?" to yes. And you must also put the code into a PHP page. (A page with the.php extension)
